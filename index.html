What Uptix does differently

One binary, one Docker container, one command. Everything included.

🔍

Built-in monitoring

HTTP, TCP, DNS, SSL expiry, cron heartbeats. No external service needed. Configurable intervals down to 30 seconds.

📋

Full incident lifecycle

Investigating → Identified → Monitoring → Resolved. Auto-create incidents from failed checks. Post updates in one click.

📧

Subscriber notifications

Email, webhook, Slack, Discord. Users subscribe once, get pushed updates for every incident. No more "is it just me?"

🔌

REST API + Webhooks

Create incidents from CI/CD. Update status from Terraform. Integrate with anything. Full OpenAPI spec.

🪶

Actually lightweight

Single Go binary. Under 30MB Docker image. Uses ~10MB RAM at idle. Runs on a $5 VPS. No Kubernetes. No PhD required.

🎨

Beautiful out of the box

Dark mode, custom logo, custom domain, custom CSS. Looks professional on day one. Your customers will trust you more.

💾

PostgreSQL or SQLite

Start with SQLite for a single server. Scale to PostgreSQL when you need HA. No migration headaches.

🔄

Recurring maintenance windows

Schedule weekly/monthly maintenance. Auto-creates and auto-resolves. No more "why is it down every Sunday at 2 AM" panic.

🤖

MCP server for AI agents

Let Claude or ChatGPT query your service status directly. The first status page built for the agent era.
